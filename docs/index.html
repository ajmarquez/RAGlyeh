Lovecraft Grounded AI Lab — Milestone roadmap (beginner-friendly)

A compressed, motivation-first learning plan: start from zero with FastAPI, get to grounded answers early, then slow down where real systems work happens (retrieval quality, latency, evals, and observability).

Shape: 6 milestones (~10-12 weeks at 4h/week) Pacing: early wins in hours, later phases intentionally slower Output: reusable grounded QA template (Lovecraft to any corpus)
What you will build: A FastAPI grounded AI systems lab over public-domain Lovecraft text: lexical/vector/hybrid retrieval, reranking, metadata-aware filtering, citation-first answering, evaluation harnesses, and request-level tracing. What this teaches: Retrieval quality, evaluation, latency, observability, and grounded answer behavior. The final result should feel closer to a small AI systems lab than a toy RAG demo. How to use this roadmap: Work through milestone checklists in order. Milestones are not equal effort; early milestones are short, later ones require more debugging and validation. The goal is depth, not tool sprawl.
Tip: your first visible wins should happen fast. Later milestones slow down because retrieval tuning, measurement, and observability are systems work.
Milestone A Foundations 0%

Goal

Get a clean local FastAPI project running from zero.

Estimated total

4-6 hours (often faster if focused).

What you will learn

  • How uvicorn app.main:app --reload maps to your app entrypoint.
  • How to keep dependencies reproducible and project layout clear.

Checklist

Definition of done

Done when /health runs locally and dependencies are reproducible on another machine.

Loose pacing

Usually week 1 or less.

Milestone B First usable API 0%

Goal

Make story ingestion/query routes usable without any database yet.

Estimated total

4-6 hours.

What you will learn

  • Pydantic models and simple API contract design.
  • Basic error handling and local text ingestion flow.

Checklist

Definition of done

Done when you can ingest a text file and retrieve its chunks.

Loose pacing

Often week 1-2 combined with Milestone A.

Milestone C Retrieval MVP (first grounded answer early) 0%

Goal

Ship a real /ask with citations and grounded behavior early.

Estimated total

6-8 hours.

What you will learn

  • Chunking strategy and stable chunk IDs as citations.
  • Lexical retrieval + grounded prompting with "don't know" behavior.

Checklist

Definition of done

Done when /ask returns grounded answers with citations and refuses unsupported claims.

Loose pacing

Usually reaches first LLM-powered answer by week 2-4.

Milestone D Persistence + hybrid retrieval + reranking 0%

Goal

Move from in-memory MVP to persistent storage, hybrid retrieval, and candidate reranking.

Estimated total

10-14 hours.

What you will learn

  • Docker Compose + Postgres only when you need persistence.
  • Why keyword/lexical search helps exact names/titles/phrases while semantic search helps fuzzy thematic matches.
  • Why production retrieval often needs lexical + vector together, not either/or.
  • Why first-pass retrieval is often not enough without a reranking step.

Checklist

Definition of done

Done when lexical, vector, and hybrid retrieval all work end-to-end, metadata filters are usable, and reranking quality gains are measured.

Loose pacing

This is where pace slows down due to infra and debugging overhead.

Milestone E Evaluation harness + observability 0%

Goal

Adopt a systems mindset: measure behavior by question type, compare changes, and debug with evidence.

Estimated total

8-12 hours.

What you will learn

  • How to separate evals into factual, comparative, interpretive, and refusal categories.
  • How to measure grounding quality, not just fluent answer quality.
  • How request IDs, traces, and latency/token metrics make failures debuggable.

Checklist

Definition of done

Done when you can run category-based evals, inspect traces, and pinpoint where grounding or latency breaks down.

Loose pacing

Later-phase effort: complexity rises because evaluation and observability are systems work.

Milestone F Demo + polish 0%

Goal

Turn the project into a clean, shareable grounded assistant demo and reusable template.

Estimated total

4-6 hours.

What you will learn

  • How to present grounded QA behavior, not just "question in, answer out".
  • How to design UX that fails gracefully under weak retrieval or ambiguous questions.

Checklist

Definition of done

Done when the demo clearly shows grounded behavior, uncertainty handling, and citations users can inspect.

Loose pacing

Usually the final 1-2 weeks. Keep provider switching secondary to demo usability.

What you can reuse after this project

  • A grounded QA API and ingestion pattern for any plain-text corpus.
  • Hybrid retrieval + reranking patterns that transfer beyond Lovecraft.
  • Metadata-aware filtering for debugging, exact retrieval, and corpus extension.
  • Eval and observability workflow for continuous grounding and latency improvement.

This roadmap starts with Lovecraft for focus and public-domain convenience, but the architecture is intentionally reusable for other corpora. The goal is not just to answer questions, but to understand where retrieval fails, how to measure it, and how to make behavior more trustworthy.
