Lovecraft RAG Engine — Step-by-step plan (4 hours/week)

A structured weekly checklist to learn FastAPI → Chunking → First LLM call → Retrieval → RAG → pgvector → Evals → Traces → Demo UI. Designed for ~2 sessions × 2 hours per week.

Pace: 4h/week Duration: 17 weeks Output: Portfolio-ready repo + reusable RAG template
How to use: Each week has a single outcome. Check items as you finish them. Your progress is saved in your browser (localStorage). Two-session rhythm: Session A: implement core change. Session B: debug + small refactor + write notes.
Tip: keep commands standardized (later): make dev, make test, make eval.

Week 3 Load real texts (local files) + story ingestion 0%

Outcome

Checklist

Keep it small

Where to get the texts

Week 4 Chunking 0%

Outcome

Checklist

Why chunk IDs matter

They become your "citations". Everything else (vectors, reranking, evals) builds on this.

Docs

  • LangChain text splitter — good reference for chunking strategies
  • Tip: add a small overlap (~150 chars) between chunks so context isn't lost at the edges
